+++ title = "Cloudflare Sunsetting Pages" date = 2025-08-12 type = "articles" tags = ["cloudflare", "workers", "pages", "migration", "web development"] +++

If you travel to the Cloudflare Pages documentation, you may notice the sparkly little link shown below on the left-hand toolbar:

That link will take you to a migration guide. While Cloudflare hasn't officially announced Pages' deprecation, their strategic direction is clear. In their recent full-stack development post:

"Cloudflare Pages will continue to be supported, but, going forward, all of our investment, optimizations, and feature work will be dedicated to improving Workers. Now that Workers supports both serving static assets and server-side rendering, you should start with Workers."

The message is clear: while Pages isn't going away immediately, Workers is the platform receiving active development. When rolling out this site, I had intended on using Pages. However, noticing all the above I went with Workers instead, despite being fully static.
