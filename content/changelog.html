How versioning works

This site uses Semantic Versioning (SemVer) to track changes. Unlike traditional software, this site combines both code and content, so the versioning reflects the overall user experience rather than just technical changes.

MAJOR versions (2.0.0)

Breaking changes that significantly alter how you experience the site:

  • • Complete visual redesigns
  • • URL structure changes (old bookmarks might break)
  • • Removing major sections or features
  • • Navigation overhauls

MINOR versions (1.3.0)

New features and functionality that enhance the site:

  • • New content sections (like the blogroll or notes)
  • • New functionality (search, RSS feeds, commenting)
  • • Significant content additions or restructuring
  • • New interactive features

PATCH versions (1.2.1)

Small improvements and fixes:

  • • Bug fixes and styling tweaks
  • • Individual blog posts or notes
  • • Typo corrections
  • • Performance improvements
  • • Minor design adjustments

All versions are tagged in the Git repository, making it easy to see exactly what changed between releases.
